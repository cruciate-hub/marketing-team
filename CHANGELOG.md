## marketing-team 13.7

Second audit pass on the `blog-seo-content` compliance gate shipped in 13.6. An adversarial review against the live blog (`website/pages-blog.json`, 258 posts) and 11 targeted fixtures surfaced eleven categories the first audit missed — false positives that fire on real published content, false negatives that slipped through the gate, and a workflow gap where the `internal-linking-strategist`'s anchor text was never validated. **Non-breaking** — same skill, same namespace, same draft→check→convert→link flow; the script gains accuracy and a new sub-command, and the skill picks up a regression suite that every future audit starts from.

- [`scripts/compliance.py`](marketing-team/skills/blog-seo-content/scripts/compliance.py): **eleven correctness fixes**. (1) **Code-block headings** — a `#` comment inside a fenced code block was counted toward `single_h1` / heading-skip; `check_headings` and `check_keyword_in_h2` now strip fenced code via a small `strip_fenced_code` helper (matches what the prose-sensitive checks already do). (2) **Heading-skip severity** — `no_skipped_heading_levels` downgraded FAIL → WARN; 50 of 258 live posts (19%) skip a level, 44 deliberately go H1→H3, so hard-FAILing the house style was blocking valid drafts. (3) **Emoji blocks** — added U+2139 (ℹ), U+2049 (⁉), U+203C (‼), U+2300–U+23FF (⏰⌚⏳⌨), U+25A0–U+25FF (▶ etc.) to `EMOJI_PATTERN`; tone.md ("Do not use emojis") was silently passing these. (4) **CommonMark autolinks** — `<https://…>`, `<mailto:…>`, and `<user@host>` are valid markdown but were flagged as raw HTML; `check_html_tags` now strips them before the tag scan. (5) **Smart-punctuation keyword match** — a curly `’` in the body vs straight `'` in the title (or vice versa, common when the HTML step applies smart punctuation) false-failed `keyword_in_first_paragraph` / `keyword_in_h2`; both checks now NFKC-normalize and curly→straight-fold via a new `normalize_quotes` helper. (6) **Alt text scanning** — the 13.6 pass folded the meta description into the scanned prose; alt text (also reader-facing, ships in the `image-alt-text` Webflow field) was still skipped, so em dashes and forbidden terms in alt text were shipping. `run()` now folds `altText` into `prose` alongside body + title + meta description. (7) **cp1252 fallback** — a Word-paste draft (cp1252-encoded em dash 0x97) crashed with an unhandled `UnicodeDecodeError`; new `_read_draft` helper tries `utf-8-sig` then falls back to `cp1252`, preserving em-dash detection (`errors='replace'` would have silently dropped the em dash). (8) **Filler-opener bypass** — `**In today's**` did not match `^In today'?s` because `first_sentence` returned raw markdown; it now strips leading `*`, `_`, `>`, and list markers before the filler check. (9) **Reading-time sanity** — `Minutes to read` was parsed but never validated; new `check_reading_time` WARNs when the claimed value drifts by more than 1 from `round(word_count/250)` (LLMs frequently paste a stale or made-up number). (10) **Leverage compound** — `\bleverag(e|ing)\b` fired on "high-leverage" / "higher-leverage" (legitimate strategy English); tightened to `(?<![-\w])leverag(e|es|ed|ing)\b` — same fix also expands coverage to "leveraged" and "leverages", which the old pattern missed entirely. (11) **Linker-anchor coverage** — the internal-linking-strategist injects `<a href>` anchors into the HTML **after** the draft.md compliance pass, so its anchor text bypassed every brand-violation check; new `--scan-text` flag and `scan_text()` function expose the em-dash / emoji / forbidden-term / risky-term checks for vetting each anchor before it is embedded.
- [`marketing-team/skills/blog-seo-content/SKILL.md`](marketing-team/skills/blog-seo-content/SKILL.md): "Internal links" section now requires `python3 scripts/compliance.py --scan-text 'anchor text'` on every linker-supplied anchor before embedding, with rejected anchors sent back for an alternative — closes the post-linker validation gap. Slug guidance gained a one-sentence note that em-dash titles must be fixed before slug derivation (em dashes collapse to `---` and the slug check rejects consecutive hyphens).
- [`marketing-team/skills/blog-seo-content/tests/`](marketing-team/skills/blog-seo-content/tests): **first committed regression suite** for the gate. Ten fixtures (`F1`–`F11`), each isolating one targeted behaviour the audit caught, plus `run_tests.py` with per-fixture expectations (`must_fail`, `must_warn`, `must_pass`, `must_not_fail`, `exit_code`) and a README documenting the suite and how to extend it. Exit 0 if every fixture behaves as documented; exit 1 with per-fixture diff on any drift. The 13.6 audit referenced a 26-fixture suite that was never committed; future audits now start from a committed baseline.
- [`docs/blog-seo-content.md`](docs/blog-seo-content.md): added the linker-anchor vet to the workflow steps so the doc tracks the SKILL.md change. [`marketing-team/README.md`](marketing-team/README.md): refreshed the `blog-seo-content` row (SKILL.md 363 → 365 lines / 21.4 → 22.0 KB).

## marketing-team 13.5 + brand-kit 3.5

Reliability hardening of the **canonical fetch block** that every content skill runs to load its reference files. The block ignored the exit code of `git pull --ff-only --quiet` and never checked that the working tree was complete, so a broken, partial, or stale local clone silently served missing or out-of-date brand files. In practice this produced confidently wrong conclusions: a session reported that `messaging/terminology.md` and `messaging/tone.md` "don't exist" and that the brand-wide em-dash ban "isn't real", when both files were present on the remote the whole time. **Non-breaking** — same skills, same namespaces, same install commands; the only behavior change is that a bad fetch now fails loud instead of silent.

- [`scripts/canonical-fetch-block-v2.md`](scripts/canonical-fetch-block-v2.md): rewrote the clone/refresh logic. (1) **Never deletes an existing `$REPO`** — the default path doubles as a working checkout that can hold un-pushed local commits, so the old clone-or-pull path is now guarded against destroying local work. (2) **A failed `pull` is surfaced**, not swallowed. (3) New **mechanical integrity gate** probes core files across `brain.md`, `messaging/`, and `design-system/`; if any are missing or empty (or `git rev-parse HEAD` fails), the block prints `Fetch failed: …` with recovery guidance and `exit 1`s, so the skill stops instead of writing copy off incomplete brand rules. The "validate every file" rule was previously prose the model could skip; it is now enforced in shell.
- Synced the updated block verbatim into all 12 fetch-using SKILL.md files via [`scripts/sync-fetch-blocks.py`](scripts/sync-fetch-blocks.py): aeo-content, backlink-placement-finder, blog-publisher, blog-seo-content, brand-messaging, case-study, design-system, internal-linking-strategist, newsletters, press-release, product-update-vs-website, site-intelligence. [`scripts/audit-skills.sh`](scripts/audit-skills.sh) reports 0 fetch-block drift across the 12.
- Bumped [`marketing-team/.claude-plugin/plugin.json`](marketing-team/.claude-plugin/plugin.json) from 13.4 to 13.5 (semver minor — non-breaking hardening).
- Bumped [`brand-kit/.claude-plugin/plugin.json`](brand-kit/.claude-plugin/plugin.json) from 3.4 to 3.5 — `brand-messaging` and `design-system` are symlinked into brand-kit, so the fetch-block fix ships in both plugins (same precedent as marketing-team 12.6 + brand-kit 3.4).

## 10.1

Hardening on the v10.0 fetch architecture — strictly additive, surfaced by Cowork cold-tests; the git-clone + cat + sed-chunks design from v10.0 is unchanged.

- [scripts/canonical-fetch-block-v2.md](scripts/canonical-fetch-block-v2.md) now covers both harness truncation-marker formats — the `Output too large (NkB). Full output saved to: …` preview form AND the `Error: result (N characters) exceeds maximum allowed tokens` sidecar-only form. v10.0 only documented the first; an LLM hitting the second could miss the truncation signal and proceed on partial output. Re-stamped across all 12 fetch-using SKILL.md files (audit drift=0).
- Calibrated the truncation threshold empirically at ~50,000 characters of JSON-wrapped tool result (~48–55 KB raw stdout depending on token density); the 250-line `sed` chunks (~17 KB each) recommended in the canonical block keep ~3× safety margin.
- [marketing-team/skills/aeo-content/scripts/duplicate_check.py](marketing-team/skills/aeo-content/scripts/duplicate_check.py) now emits a dual signal — exit code AND a final stdout line: `RESULT: MATCHES` (exit 1), `RESULT: CLEAN` (exit 0), `RESULT: UNVERIFIED` (exit 2). [aeo-content/SKILL.md](marketing-team/skills/aeo-content/SKILL.md) documents both paths and tells the calling LLM to treat any disagreement between exit code and stdout marker as UNVERIFIED.
- Bumped [.claude-plugin/marketplace.json](.claude-plugin/marketplace.json) and [marketing-team/.claude-plugin/plugin.json](marketing-team/.claude-plugin/plugin.json) from 10.0 to 10.1.

## 10.0

New fetch architecture — every skill that loads reference content now does a single shallow `git clone` per session into a local /tmp folder, then reads files with `cat` (or `sed` chunks for outliers) instead of fetching per-read over HTTP.

- Replaced the WebFetch + `github.com/.../blob/…` model, which returned 40–55× HTML overhead per file, occasionally tripped GitHub's anonymous abuse-detection on parallel fetches (newsletters fetches 11), and silently rewrote brand-critical strings via WebFetch's internal LLM (e.g. `social.plus` → `Social.plus`).
- Confirmed empirically that every clean-content alternative is blocked in Cowork — `raw.githubusercontent.com`, `cdn.jsdelivr.net`, `api.github.com`, and `codeload.github.com` all return 403; only `github.com` is reachable, which is what made shallow clone the answer. Reviewed by Gemini and a separate Claude session, both converged on the same recommendation.
- Added [scripts/canonical-fetch-block-v2.md](scripts/canonical-fetch-block-v2.md) as the single source of truth for the fetch instructions and [scripts/audit-skills.sh](scripts/audit-skills.sh) to enforce byte-identity across all 12 fetch-using SKILL.md files.
- One clone per session into `/tmp/cruciate-hub-marketing-team`; `cat` for files that fit, 250-line `sed` chunks when the harness's truncation marker fires, halve-and-retry if a chunk itself truncates; `python3`/`jq` for large JSON inventories. Hard-fail on any fetch error with `Fetch failed: <path>. Please check your network connection and rerun.` — no reconstruction from memory, no WebFetch fallback.
- Removed `skills/skills/aeo-content/scripts/fetch_brand.py` (the fragile HTML-parsing helper) and rewrote [duplicate_check.py](marketing-team/skills/aeo-content/scripts/duplicate_check.py) to read directly from `$MT_REPO/website/pages-*.json` instead of fetching at runtime.
- Dropped the stale "raw is blocked" warning copy-pasted across three `brain.md` files; corrected the architecture description in 12 README files.
- Fallout from the v10.0 verification battery (test D1 — "brain.md mentions all 15 skills"): added `internal-linking-optimizer`, `legal-docs-formatter`, and `svg-icon-transformer` to [brain.md](brain.md)'s Available skills table; three skills had been added in earlier commits but never reached the master router.
- Bumped [.claude-plugin/marketplace.json](.claude-plugin/marketplace.json) and [marketing-team/.claude-plugin/plugin.json](marketing-team/.claude-plugin/plugin.json) from 0.9.2 to 10.0.
