Second audit pass on the `blog-seo-content` compliance gate shipped in 13.6. An adversarial review against the live blog (`website/pages-blog.json`, 258 posts) and 11 targeted fixtures surfaced eleven categories the first audit missed — false positives that fire on real published content, false negatives that slipped through the gate, and a workflow gap where the `internal-linking-strategist`'s anchor text was never validated. **Non-breaking** — same skill, same namespace, same draft→check→convert→link flow; the script gains accuracy and a new sub-command, and the skill picks up a regression suite that every future audit starts from.

- [`scripts/compliance.py`](marketing-team/skills/blog-seo-content/scripts/compliance.py): **eleven correctness fixes**. (1) **Code-block headings** — a `#` comment inside a fenced code block was counted toward `single_h1` / heading-skip; `check_headings` and `check_keyword_in_h2` now strip fenced code via a small `strip_fenced_code` helper (matches what the prose-sensitive checks already do). (2) **Heading-skip severity** — `no_skipped_heading_levels` downgraded FAIL → WARN; 50 of 258 live posts (19%) skip a level, 44 deliberately go H1→H3, so hard-FAILing the house style was blocking valid drafts. (3) **Emoji blocks** — added U+2139 (ℹ), U+2049 (⁉), U+203C (‼), U+2300–U+23FF (⏰⌚⏳⌨), U+25A0–U+25FF (▶ etc.) to `EMOJI_PATTERN`; tone.md ("Do not use emojis") was silently passing these. (4) **CommonMark autolinks** — `<https://…>`, `<mailto:…>`, and `<user@host>` are valid markdown but were flagged as raw HTML; `check_html_tags` now strips them before the tag scan. (5) **Smart-punctuation keyword match** — a curly `’` in the body vs straight `'` in the title (or vice versa, common when the HTML step applies smart punctuation) false-failed `keyword_in_first_paragraph` / `keyword_in_h2`; both checks now NFKC-normalize and curly→straight-fold via a new `normalize_quotes` helper. (6) **Alt text scanning** — the 13.6 pass folded the meta description into the scanned prose; alt text (also reader-facing, ships in the `image-alt-text` Webflow field) was still skipped, so em dashes and forbidden terms in alt text were shipping. `run()` now folds `altText` into `prose` alongside body + title + meta description. (7) **cp1252 fallback** — a Word-paste draft (cp1252-encoded em dash 0x97) crashed with an unhandled `UnicodeDecodeError`; new `_read_draft` helper tries `utf-8-sig` then falls back to `cp1252`, preserving em-dash detection (`errors='replace'` would have silently dropped the em dash). (8) **Filler-opener bypass** — `**In today's**` did not match `^In today'?s` because `first_sentence` returned raw markdown; it now strips leading `*`, `_`, `>`, and list markers before the filler check. (9) **Reading-time sanity** — `Minutes to read` was parsed but never validated; new `check_reading_time` WARNs when the claimed value drifts by more than 1 from `round(word_count/250)` (LLMs frequently paste a stale or made-up number). (10) **Leverage compound** — `\bleverag(e|ing)\b` fired on "high-leverage" / "higher-leverage" (legitimate strategy English); tightened to `(?<![-\w])leverag(e|es|ed|ing)\b` — same fix also expands coverage to "leveraged" and "leverages", which the old pattern missed entirely. (11) **Linker-anchor coverage** — the internal-linking-strategist injects `<a href>` anchors into the HTML **after** the draft.md compliance pass, so its anchor text bypassed every brand-violation check; new `--scan-text` flag and `scan_text()` function expose the em-dash / emoji / forbidden-term / risky-term checks for vetting each anchor before it is embedded.

## 10.0

New fetch architecture — every skill that loads reference content now does a single shallow `git clone` per session into a local /tmp folder, then reads files with `cat` (or `sed` chunks for outliers) instead of fetching per-read over HTTP.

- Replaced the WebFetch + `github.com/.../blob/…` model, which returned 40–55× HTML overhead per file, occasionally tripped GitHub's anonymous abuse-detection on parallel fetches (newsletters fetches 11), and silently rewrote brand-critical strings via WebFetch's internal LLM (e.g. `social.plus` → `Social.plus`).
- Confirmed empirically that every clean-content alternative is blocked in Cowork — `raw.githubusercontent.com`, `cdn.jsdelivr.net`, `api.github.com`, and `codeload.github.com` all return 403; only `github.com` is reachable, which is what made shallow clone the answer. Reviewed by Gemini and a separate Claude session, both converged on the same recommendation.
- Added [scripts/canonical-fetch-block-v2.md](scripts/canonical-fetch-block-v2.md) as the single source of truth for the fetch instructions and [scripts/audit-skills.sh](scripts/audit-skills.sh) to enforce byte-identity across all 12 fetch-using SKILL.md files.
- One clone per session into `/tmp/cruciate-hub-marketing-team`; `cat` for files that fit, 250-line `sed` chunks when the harness's truncation marker fires, halve-and-retry if a chunk itself truncates; `python3`/`jq` for large JSON inventories. Hard-fail on any fetch error with `Fetch failed: <path>. Please check your network connection and rerun.` — no reconstruction from memory, no WebFetch fallback.
- Removed `skills/skills/aeo-content/scripts/fetch_brand.py` (the fragile HTML-parsing helper) and rewrote [duplicate_check.py](marketing-team/skills/aeo-content/scripts/duplicate_check.py) to read directly from `$MT_REPO/website/pages-*.json` instead of fetching at runtime.
